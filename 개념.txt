제로 트러스트 
모든 네트워크 경계를 신뢰하지않는다.
네트워크 접속을 하러면 먼저 인증을 받아야한다


실드게이트
제로트러스트 업무 환경을 구현하기 위해서 사용하는 기술
IAP+RBI 기술을 사용한다.


IAP
특정 애플리케이션이나 리소스에 대한 안전한 엑세스를 제공하기 위한 서비스이다.

사용자가 애플리케이션을 엑세스하려 할 때, IAP는 사용자의 ID와 함께 다른 보안 속성을 확인하여 해당 사용자에게 적절한 
권한이 있는지 확인한다.

VPN을 사용하지 않고도 신뢰할 수 없는 네트워크에서 업무 처리(VPN 필요없음)

관리자 IAP를 이용하여 세밀한 엑세스 규칙 설정 가능 
예) 특정 사용자만 애플리케이션 접속 가능 (세밀한 엑세스 제어)

클라우드 기반의 애플리케이션 뿐만 아니라 온프레미스 애플리케이션에 대해서도 안전한 엑세스 제공

모든 엑세스 요청은 로깅, 추후 보안 감사나 이상 징후 탐지에 사용될 수 있습니다. (로깅 및 감사 기능)

제로 트러스트 엑세스 모델 구현


RBI

웹 브라우징 활동을 사용자의 장치나 로컬 네트워크에서 격리시키고, 
중앙 집중화된 환경에서 웹 콘텐츠를 렌더링하고 해석하는 것입니다.


사용자의 장치에서 웹 콘텐츠를 직접 렌더링하는 대신, 중앙 집중화된 서버에서 웹 사이트를 렌더링하고 
사용자에게 결과를 전송합니다. 이렇게 하면 악성 코드나 악성 웹사이트로부터 사용자의 장치를 격리시키고
보안을 강화할 수 있습니다.


네트워크 세분화
네트워크를 더 작고 개별적인 하위 네트워크로 분할하여 각 하위네트워크에 고유한 보안 제어 및 서비스를 제공할수 있도록
지원하는 네트워크 보안 기술

각 네트워크 별로 보안 서비스를 제공해서 네트워크 트래픽을 더 효과적으로 제어하고, 네트워크 성능을 향상, 보안태세를 개선

네트워크가 분할되면 하위네트워크 내에서 네트워크 트랙픽이 격리되어 공격 범위가 줄어들고 수평 이동이 지연됩니다.

수평이동이 지연되서 경계가 침해될경우 네트워크 전체에서 횡방향으로 이동하지 못하도록 방지한다.
